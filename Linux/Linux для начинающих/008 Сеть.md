## IP a
Локальный адрес
```
[admin@igonin-vl share]$ ip a
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000
    inet 127.0.0.1/8 scope host lo
2: enp0s3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast state UP group default qlen 1000
    inet 10.0.2.15/24 brd 10.0.2.255 scope global noprefixroute dynamic enp0s3
```
## IP r
Маршрутизация
```
[admin@igonin-vl share]$ ip r
# внешняя маршрутизация
default via 10.0.2.2 dev enp0s3 proto dhcp metric 100
# маршрутизация внутри сети через сам локальный 10.0.2.15
10.0.2.0/24 dev enp0s3 proto kernel scope link src 10.0.2.15 metric 100 
```
## etc resolve_conf
Dns 
```
[admin@igonin-vl share]$ cat /etc/resolv.conf
# Generated by NetworkManager
search ad.speechpro.com
nameserver 10.12.10.10
nameserver 10.12.10.20

# пинг до общественного dns сервера google (есть веде, где есть физический выход в internet)
# если пинг сюда не проходит - проблема с физикой
[admin@igonin-vl share]$ ping 8.8.8.8
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=106 time=7.07 ms

# если проходит, но при этом не видит никаких internet-ресурсов, то проблема с отсутствием (в /etc/resolv.conf) или корректностью настройки nameserver
```
